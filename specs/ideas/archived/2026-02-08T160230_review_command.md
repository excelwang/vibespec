# Idea: Vibespec Review Command

**Timestamp**: 2026-02-08T16:02:30+08:00

## Proposal

Add `vibespec review [SPEC_ID]` command to facilitate focused reviews.

**Workflow**:
1. User runs `vibespec review [id]`.
2. Agent loads the target spec and its parent/child dependencies.
3. Agent performs **Review Protocol** checks (HIERARCHY, REDUNDANCY, CONTRADICTION, FOCUS).
4. Agent generates a structured review report.

## Rationale

- **Focus**: Allows targeted quality control on specific areas.
- **On-Demand**: decouples review from the automate/refine loop.
- **Human-in-the-Loop**: Provides a tool for manual auditors to get AI assistance.

## Target

- **SKILL**: Add `vibespec review` trigger.
- **L2**: Update `REVIEWER` role to support on-demand invocation.
- **CLI**: Add argument handling.
