# Gate Workflows Reference

> **Load when**: A `vibespec fix gate` or `vibespec triage gate` trigger fires and the Agent needs the phase prompt for the unified gate.

## Routing

- Triage: `UnifiedGateTriageWorkflow.DetectAndPlan`
- Fix: `UnifiedGateFixWorkflow.ExecuteRepairPlan`

## Unified Gate

### Triage Phase

**Workflow**: `UnifiedGateTriageWorkflow.DetectAndPlan`

Prompt:

```text
Execute the unified triage gate phase: Detect + Plan.

Requirements:
1. Audit the latest frozen baseline or submission in strict priority order:
   - `spec-drift`
   - `src-drift`
   - `quality`
2. Do not repair anything in this phase.
3. After each defect class is classified, immediately publish that batch through scripts/agent_sync.py.
4. If the published batch contains defects, release the waiting fix session without waiting for later classes to finish.
5. For every defect, generate:
   - stable ID
   - defect type
   - evidence
   - summary
   - explicit repair logic for the fix session
6. Accept only when no defects remain after all three classes are classified.
```

### Fix Phase

**Workflow**: `UnifiedGateFixWorkflow.ExecuteRepairPlan`

Prompt:

```text
Execute the unified fix gate phase: Execute Repair Plan.

Requirements:
1. If no released repair work exists yet, wait on the fix gate lock.
2. Load only the latest frozen repair items generated by triage.
3. Treat the latest triage-generated repair plan as the scope boundary for the current execution cycle.
4. If multi-round repair is needed, create fresh run artifacts under `specs/build/<timestamp>/`, including `todo.md` and `auto-decisions.md`.
5. Generate repair todo items from structural and semantic findings within the released scope only; do not invent new repair goals outside that scope.
6. Ground auto-decisions in the triage repair logic, released scope, relevant upper-layer contracts, and the latest validation or re-scan evidence.
7. If multiple repair options remain, prioritize scope preservation, traceability preservation, and the smallest safe behavioral delta.
8. Continue autonomously while actionable repair items remain, rerunning validation and relevant tests after each round and regenerating the todo from fresh findings.
9. Log every auto-decision with timestamp, context, options considered, chosen option, rationale, and affected files or spec IDs.
10. Triage may still be classifying later defect classes while fix starts on earlier released work.
11. Publish a frozen submission only after triage completes the full classification cycle and hands off final turn ownership.
12. Validate the resulting changes.
13. Publish a new frozen submission through scripts/agent_sync.py with a response for every repair item.
```
